animCJK SVG の仕組み

ひらがな「あ」(U+3042) を例に、animCJK が書き順アニメーション SVG をどう構成しているかを見ていきます。

1. 完成した姿

まずは完成した SVG をそのまま再生してみましょう。Replay を押すと書き順通りに描かれます。

完成形（3 画）

2. ファイル全体の構成

animCJK の SVG は「5 つの要素」で構成されています。

<!-- ① ライセンスコメント -->
<svg id="z12354" class="acjk" viewBox="0 0 1024 1024">

  ② スタイル + キーフレーム定義
  <style>
    @keyframes zk { to { stroke-dashoffset: 0; } }
    svg.acjk path[clip-path] { ...アニメ設定... }
    svg.acjk path[id] { fill: #ccc; }
  </style>

  ③ シルエット（各画の塗り部分）
  <path id="z12354d1"  d="M660 211 ..."/>   ← 1画目（横棒）
  <path id="z12354d2"  d="M334 128 ..."/>   ← 2画目（縦棒+曲線）
  <path id="z12354d3a" d="M559 431 ..."/>   ← 3画目（前半）
  <path id="z12354d3b" d="M602 484 ..."/>   ← 3画目（後半）

  ④ シルエットを clip-path 用マスクに変換
  <defs>
    <clipPath id="z12354c1"><use href="#z12354d1"/></clipPath>
    ... (d2, d3a, d3b も同様)
  </defs>

  ⑤ 中心線（median）— これがアニメで「描かれる」
  <path style="--d:1s" clip-path="url(#z12354c1)"  d="M 174,258 ..."/>
  <path style="--d:2s" clip-path="url(#z12354c2)"  d="M 331,137 ..."/>
  <path style="--d:3s" clip-path="url(#z12354c3a)" d="M 570,440 ..."/>
  <path style="--d:3s" clip-path="url(#z12354c3b)" d="M -170,442 ..."/>

</svg>

3. パーツを分解して見る

構成要素を視覚的に分解します。

③ シルエットだけ
（灰色のグリフ）
⑤ 中心線（median）だけ
赤線で表示
中心線を stroke-width:128 で描画
（クリップなし＝青）

左から右へ「太い線で中心線をなぞる → シルエット形状に切り抜く → 字形が書き順通り埋まる」という仕組みが分かります。

4. アニメーションの仕組み

animCJK のアニメは SVG の stroke-dashoffset アニメで実装されています。

svg.acjk path[clip-path] {
  /* 線の長さ分の破線パターン */
  stroke-dasharray: 3337;
  /* 初期は破線オフセットを線の長さ以上にして「全部消した」状態 */
  stroke-dashoffset: 3339;

  /* これを 0 までアニメすると「先端から線が伸びる」 */
  animation: zk var(--t) linear forwards var(--d);
}
@keyframes zk { to { stroke-dashoffset: 0; } }
pathLength="3333" の役割: 中心線の実長は数字によって違いますが pathLength 属性で「このパスは長さ 3333 ということにする」と仮想的に正規化できます。だから stroke-dasharray: 3337 は全パスで使えます。

中心線は stroke-width: 128 で描画されます。1024×1024 のキャンバス上で字形のシルエットを覆う太さです。さらに clip-path でシルエットの形に切り抜くため、太線が動いているのに字形が書き順通りに埋まるように見えます。

--d による書き順制御

各画の median には --d:Ns という CSS 変数が付いていて、これが animation-delay になります。

画--d役割
1画目--d: 1s1秒後にアニメ開始
2画目--d: 2s2秒後にアニメ開始
3画目--d: 3s3秒後にアニメ開始

各アニメの長さは --t: 0.8s なので、書き順通りに 1秒間隔で次々と描かれていきます。

5. 3画目の分割テクニック

「あ」の 3画目はループする複雑な形状です。シルエットを 2 つに分け、両方のアニメを同時開始することで 1 つの連続ストロークがループを描いているように表示します。

3画目 前半 (d3a)
右側の C 字部分
3画目 後半 (d3b)
下のループ部分

1ストロークに見える

後半 (d3b) の median は M -170,442 ... とキャンバス外から始まります。クリップでループの内側部分しか見えないため、線がループを横切って visible になる効果になります。

6. animNumber との対応

animNumber (main.go) はこの構造を Go コードで生成しています。 buildSVG 関数の処理が animCJK の各要素に対応します。 

// main.go:240 あたり
sb.WriteString(`<svg id="..." class="acjk" viewBox="0 0 1024 1024" ...>`)
sb.WriteString(svgStyle)              // ② style ブロック

for ... {
  <path id="..." d="..."/>            // ③ Stroke の Outline
}

<defs>
  <clipPath> ... </clipPath>            // ④ clipPath 化
</defs>

for ... {
  <path style="--d:Ns" clip-path=".." pathLength="3333" d="..."/>
                                       // ⑤ Median を polyline (M..L..) で出力
}

main.go の DigitDef / StrokeDef 構造が Outline と Median に対応します。Median は []Point で手書き定義され、 buildMedianPath が M x y L x y L x y ... 形式に展開します。

まとめ

要素役割

<clipPath>シルエットを切り抜きマスクに変換
<path clip-path>太線（128px）の中心線。stroke-dashoffset アニメで描画される
